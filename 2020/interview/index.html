计算机基础

操作系统

系统调用

  • 异常和中断的区别

异常：cpu指令内部的事件，比如除零异常，缺页异常
中断：cpu指令之外的事件，比如设备请求中断

  • 进程与线程的含义及其区别
  • 管程

由一组数据以及定义在这组数据之上的对这组数据操作组成的软件模块（数据+操作）

  • 进程之间的通信方式

管道、消息队列、信号量、共享存储、SOCKET套接字

  • 进程之间的同步

同步机制的原则：空闲让进、忙则等待、有限等待以及让权等待
同步方式：原子操作、信号量机制（PV操作）

  • 线程同步的方式有哪些
  1. 互斥量：采用互斥对象机制，保证公共资源不会被多个线程同时访问
  2. 信号量：控制当前资源的剩余数量，控制同一时刻访问某资源的最大线程数量
  3. 事件（信号）：通过通知操作的方式实现多线程同步
  • 什么是缓冲区溢出攻击？有什么危害？其原因是什么？

含义：缓冲区溢出是指计算机向缓冲区填充数据时超出了缓冲区本身的容量
危害：溢出的数据会覆盖在合法数据上，攻击时可能会跳转并且执行一段恶意代码
原因：程序没有仔细检查用户输入

  • 什么是死锁，死锁产生的条件
  1. 含义：你拿了一部分，我拿了一部分，但双方都没有拿到运行时需要的资源数目不肯让步，从而发生死锁
  2. 条件：互斥条件、请求保持条件、不剥夺条件、循环等待条件（即，一个资源同时只能给一个进程使用，每个进程得到资源后都会一直持有直到完成任务释放资源，就算该进程被阻塞时也不会放弃原有资源，并且进程之间互相不能剥夺对方已有的资源。最终，当形成了循环等待链时，就发生了死锁。）
  3. 解锁方法：破坏死锁产生条件之一即可
  4. 避免方法：银行家算法（核心思想：将剩余资源优先分配给那些能够完成任务的进程）

就绪状态、运行状态、阻塞状态

动态分区分配的算法：首次适应算法、最佳适应算法、最坏适应算法

  • 基本分页存储管理方式

慢表 + 快表，多级页表

  • 页面置换算法（时间局部性 & 空间局部性）
  1. 先进先出算法
  2. 最近最久未使用算法（时间）
  3. 最少使用（次数）
  4. 时钟clock算法(NRU算法) -> 改进型clock算法
  • 基本分段储存管理方式

二维地址(段号+段内地址)

  • 分页和分段的区别
  1. 段是信息的逻辑单位，根据用户需求划分，因此段对用户是可见的；而页是信息的物理单位，为了管理内存方便而划分，对用户是透明的
  2. 段的大小不固定，根据功能需求而定；页的大小固定，由操作系统决定
  3. 段易于存储和保护用户的共享；页的保护和共享受到限制

计算机网络

  • 简述OSI七层模型

物理层（比特）->数据链路层（帧）->网络层（数据报）->传输层（报文段TCP或者用户数据段UDP）->会话层->表示层->应用层。
其中，在端系统中含有上述七层，而在网络传输过程的中继系统中仅含有物理层、数据链路层以及网络层三部分，有些甚至连网络层也没有

  • 简述TCP/IP四层模型

网络接口层 -> 网际层 -> 运输层 -> 应用层

  • 协议与端口的区别

端口：同一主机中，各层之间是通过端口进行数据传递的
协议：不同主机中，同一层之间是通过协议进行数据解析和编码的

  • 简述TCP与UDP的区别

两者都是OSI模型中运输层的协议。
TCP：传输控制协议，提供的是面向连接、可靠的字节流服务。传输数据之前需要先进行连接，并且提供数据验证，用于可靠的通信传输。（超时重传、丢失重复数据、流量控制），通过协议中的顺序编号和确认号实现上述功能
UDP：用户数据报协议，面向数据报的运输层协议，不需要提前连接，不对数据进行验证，传输速度更快，借助其他层进行数据校验

  • 了解的端口及其对应的服务

21 – FTP文件传输协议
22 – SSH
25 – SMTP简单邮件传输协议
80 – HTTP超文本传输协议
110 – POP3邮件协议

  • TCP的三次握手和四次分手

由于TCP提供可靠的数据传输，因此客户端与服务器必须互相确认。
三次握手：客户端向服务器发送连接建立请求（第一次握手），服务器收到后返回一个确认信号+请求信号（第二次握手，此时服务器端到客户端的连接被建立了），然后，客户端返回一个确认建立的信号（第三次握手，此时客户端到服务器端的连接被建立了）。建立连接时两个方向一般是同时的，所以可以在第二次握手时，同时传递确认和请求信号。
四次分手：分手要谨慎！！！这个理解相对简单，因为数据传输是双向的，并且两个方向的连接也不一定同时断开。所以每一个方向的传输都要一来一回进行分手确认。

  • 在浏览器中输入网址之后的执行过程
  1. 查找域名对应的IP地址（这步会依次查找浏览器缓存->系统缓存->路由器缓存->ISPNDS缓存->根域名服务器）
  2. 浏览器向IP对应的web服务器发送一个HTTP请求
  3. 服务器响应请求，发回网页内容
  4. 浏览器解析网页内容
  • HTTP中GET和POST的区别

POST请求可能修改服务器上资源
GET用于信息获取

  • 统一资源定位符URL的格式

<协议>://<主机>:<端口>/<路径>

数据库

  • 存储过程

存储过程是一个预编译的代码块，可以直接调用，用于实现一些功能，执行效率高。一个存储过程替代大量SQL语句，可以降低网络通信量，提高通信效率，并且在一定程度上确保了数据安全。

  • 索引

索引是对数据库表中一或多个列的值进行排序的结构，帮助数据库获取数据的数据结构。
索引类型：普通索引、唯一索引、主键索引、全文索引。
索引采用的数据结构通常是B-Tree或者B+Tree

  • 事务

是数据库并发操作的基本单位。一个事务是一个操作序列，这些操作是不可分割的工作单位，也可以视为一个原子操作（即执行过程中不能被打断）。另外，事务是维护数据一致性的单位。
事务的4个基本特征：原子性（事务的操作要不都成功，要不都失败）、一致性（只有合法的数据可以被写入数据库，否则应该支持回滚操作）、隔离性（允许多个用户并发访问，各个事务之间的修改必须相互独立）、持久性（事务结束，结果必须是确定和持久的）。

  • 数据库的乐观锁和悲观锁

乐观锁：乐观并发控制，假设不会发生并发冲突，只在提交操作时检查是否违反数据完整性。
悲观锁：悲观并发控制，假定会发生冲突，屏蔽一切可能违反数据完整性的操作。

  • 超键、候选键、主键和外键的区别

超键：在关系中能唯一标识元组的属性集称为关系模式的超键。一个属性可以为作为一个超键，多个属性组合在一起也可以作为一个超键。超键包含候选键和主键
候选键：是最小超键，即没有冗余元素的超键
主键：数据库表中对储存数据对象予以唯一和完整标识的数据列或属性的组合。一个数据列只能有一个主键，且主键的取值不能缺失，即不能为空值（Null）
外键：在一个表中存在的另一个表的主键称此表的外键

  • 视图及其应用场景

视图：虚拟表，是多个物理表行或列的子集。视图可以进行增删改查的操作，对它的修改不会对物理表产生影响。
应用场景：将部分数据提供给访问者；统一各个表的查询方式等

  • 删除操作drop，delete，truncate的区别
  1. delete和truncate只删除表的数据不删除表的结构
  2. delete语句是dml,这个操作会放到rollback segement中,事务提交之后才生效;如果有相应的trigger,执行的时候将被触发. truncate,drop是ddl, 操作立即生效,原数据不放到rollback segment中,不能回滚. 操作不触发trigger.
  3. 一般地，不再需要一张表的时候，用drop；想删除部分数据行时候，用delete；保留表而删除所有数据的时候用truncate
  • 数据库的范式

第一范式：所有列属性不可再分，所有的数据表都应该符合此要求
第二范式：消除了非主属性对于码的部分函数依赖（即避免了只有某几列值发生变化，其他非主属性的列值都不怎么变，避免数据冗余的问题）
第三范式：消除了非主属性对于码的传递函数依赖（基本解决了数据冗余过大、插入异常、修改异常、删除异常等问题）
BCNF范式：消除主属性对于码的部分与传递函数依赖

编程语言

C / C++

  • C语言中的malloc和C++中的new的区别
  1. new与delete对应，malloc与free对应
  2. new、delete是操作符，可以重载，只能在C++中使用
  3. malloc、free是函数，可以覆盖，C和C++中都可以使用
  4. new可以调用对象的构造函数，delete可以调用响应的析构函数
  5. malloc仅用于分配内容，free用于回收内存，并不执行构造和析构函数
  6. new、delete返回的是某种数据类型指针，malloc、free返回的是void指针
  • 面向对象的三大特性

封装、继承、多态

  • 拷贝构造函数和赋值运算符的区别
  1. 拷贝构造函数生成新的类对象，而赋值运算符不会
  2. 由于拷贝构造函数直接构造新对象，因此在初始化对象前不用检验源对象与新建对象是否相同。而赋值运算符就需要这个过程。
  • 简述C++中多态的实现原理

多态：指通过基类指针或者引用调用一个虚函数时，编译时不确定调用的时基类还是派生类的函数，运行时才确定。这样做，可以隐藏实现细节，使得代码更加模块化；并且使得接口重用，使得在类的继承和派生中，保证家族中任一类的实例的某一属性时的正确调用。

多态的实现是通过虚函数表完成的，每一个有虚函数的类（包括其派生类）都有一个虚函数表，该类的任何实例化对象中都会存放指向虚函数表的指针。该虚函数表中列出了该类的全部虚函数地址。查表的过程参见文章C++虚函数表（多态的实现原理）

  • C中static的作用
  1. 隐藏。 当我们同时编译多个文件时，所有未加static前缀的全局变量和函数都具有全局可见性，故使用static在不同的文件中定义同名函数和同名变量，而不必担心命名冲突。
  2. 持久。存储在静态数据区的变量会在程序刚开始运行时就完成初始化，也是唯一的一次初始化。共有两种变量存储在静态存储区：全局变量和static变量。
  3. 默认初始化为0。其实全局变量也具备这一属性，因为全局变量也存储在静态数据区。在静态数据区，内存中所有的字节默认值都是0×00
  • C语言中volatile的含义是什么

volatile的本意是“易变的” 。 由于访问寄存器的速度要快过RAM,所以编译器一般都会作减少存取外部RAM的优化，但有可能会读脏数据。当要求使用volatile 声明的变量的值的时候，系统总是重新从它所在的内存读取数据，即使它前面的指令刚刚从该处读取过数据。而且读取的数据立刻被保存。 精确地说就是，优化器在用到这个变量时必须每次都小心地重新读取这个变量的值，而不是使用保存在寄存器里的备份。From: 一面阿里云C++面试题汇总

  • const与#define相比的优势
  1. const常量有数据类型，而宏常量没有数据类型，编译器可以对前者进行类型安全检查。而对后者只进行字符替换，没有类型安全检查，并且在字符替换可能会产生意料不到的错误（边际效应）。
  2. 有些集成化的调试工具可以对 const 常量进行调试，但是不能对宏常量进行调试。
  • 基类的析构函数不是虚函数，会带来什么问题

派生类的析构函数用不上，会造成资源的泄漏。

  • is 和 == 的区别

is表示的是对象标示符，检查对象的内存地址是否一样。
==用于检查两个对象的值是否相等。

  • python中的可变对象与不可变对象

可变对象：创建后可以改变，但地址不会改变，比如dict, list
不可变对象：创建后就不能改变，如果改变将会指向一个新的对象，比如str, int, tuple, float

可变对象

a = [1,2,3] # id(a) -> 140455165822688
a += [4,5] # id(a) -> 140455165822688

不可变对象

a = 1 # id(a) -> 93883717583648
a = a + 1 # id(a) -> 93883717583680

默认参数引发的一个小问题

class group():
    def __init__(self, num, members=[]):
        self.num = num
        self.members = members
    
    def addMember(self, mm):
        self.members.append(mm)
g1 = group(1)
g1.addMember('1')
g2 = group(2)
g2.addMember('2')
print(g1.members) # ['1', '2']

g2的修改居然会影响到g1！！！
这是因为members是一个默认list参数，如果实例化对象时没有新建该默认参数，则所有实例化成员会共用此默认参数值。如果是不可变对象就没有这个问题了。

class group():
    def __init__(self, num, members=[]):
        self.num = num
        self.members = members
    
    def addMember(self, mm):
        self.members.append(mm)
g1 = group(1, [])
g1.addMember('1')
g2 = group(2, [])
g2.addMember('2')
print(g1.members) # ['1']

这样就正常了。但是使用默认参数的本身目的就丢失了，所以默认参数如果是可变对象，还是要非常注意的。或者说这种情况下的默认参数就没有意义了。

  • 字符串拼接中join和+的区别

数值结果上，两者是一样的，但性能上，join优于+。因为字符串是不可变对象，因此当用操作符+连接字符串的时候，每执行一次+都会申请一块新的内存，然后复制上一个+操作的结果和本次操作的右操作符到这块内存空间，因此用+连接字符串的时候会涉及好几次内存申请和复制。而join在连接字符串的时候，会先计算需要多大的内存存放结果，然后一次性申请所需内存并将字符串复制过去，这是为什么join的性能优于+的原因。

数学

  • 某城市发生了一起汽车撞人逃跑事件，该城市只有两种颜色的车，蓝20%绿80%，事发时现场有一个目击者，他指证是蓝车，但是根据专家在现场分析，当时那种条件能看正确的可能性是80%，那么，肇事的车是蓝车的概率是多少？

肇事车颜色是蓝色A1的概率为P(A1)=0.2，是绿色A2概率为P(A2)=0.8;‘+’表示看到的是蓝色，‘-’表示看到的是绿色，
《目击者看清》为条件概率，即P(+/A1)=0.8，P(-/A2)=0.8，同样的P(-/A1)=0.2，P(+/A2)=0.2；
问题是求看到蓝色车的前提下是蓝色车的概率，利用贝叶斯公式
P(A1/+)=P(+/A1)P(A1)/(P(+/A1)P(A1)+P(+/A2)P(A2))=0.80.2/(0.80.2+0.20.8)=0.5

  • 老王有两个孩子，已知至少有一个孩子是在星期二出生的男孩。问：两个孩子都是男孩的概率是多大？

男，周二：1/14
男，非周二：6/14
女，周二：1/14
女，非周二：6/14
两个孩子至少一个是周二生的男孩：1/14+1/14-1/141/14=27/196
两个孩子都是男孩且至少一个是周二生的：1/141/14+1/146/14+6/141/14=13/196
条件概率：13/27

  • 现在有一个程序由A，B两个同学结对编程完成，在整个程序中的代码比例是3:5，据往常的统计A同学的Bug率为0.01%，B同学的Bug率为0.015%，现在在改程序中发现了一个BUG，那么是由A同学的代码引起的BUG概率是

记发现BUG为事件BUG，A编写代码为事件A，B编写代码为事件B，则
A编写的代码中发现BUG的概率为P(BUG|A)=0.01%
B编写的代码中发现BUG的概率为P(BUG|B)=0.015%
对于已经编好的一个程序，P(A)=3/8，P(B)=5/8
根据贝叶斯公式得：
P(A|BUG)=P(A)P(BUG|A)/{P(A)P(BUG|A)+P(B)P(BUG|B)}=3/80.01%/(3/80.01%+5/8*0.015%) = 28.6%

  • 三个骰子摇到的点数之和为（）的概率最大？

杨辉三角类的分布，中间值最大
三个骰子：3～18，中间值为10和11，故10和11的概率最大

  • 有1,2,3,…无穷个格子，你从1号格子出发，每次1/2概率向前跳一格，1/2概率向前跳两格，走到格子编号为4的倍数时结束，结束时期望走的步数为

则从1号开始走，我们的目标是求f(1)
f(1) = 0.5 * ( 1 + f(2) ) + 0.5 * ( 1 +f(3)) 即有0.5概率走一步到2号，0.5概率走两步到3号
f(2) = 0.5 * ( 1 + f(3) ) + 0.5 * ( 1 +f(4)) 即有0.5概率走一步到3号，0.5概率走两步到4号(结束)
f(3) = 0.5 * ( 1 + f(4) ) + 0.5 * (1 + f(1)) 即有0.5概率走一步到4号，0.5概率走两步到5号(5号即可看做1号)
f(4) =0 走到4号就结束了，故为0
可以解上述方程，得f(1) = 18/5.
