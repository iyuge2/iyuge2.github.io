Python中日志模块的简易用法

发表于2021-01-25|更新于2021-03-01|技术区
|字数总计:462|阅读时长:2分钟|阅读量:

写在前面

本文仅提供python中logging模块的简易用法，若需更详细的教程，可以参见官方文档。

记录等级

  • 日志重要性级别 DEBUG < INFO < WARNING < ERROR < CRITICAL
  • 默认的记录级别是WARNING，表示仅追踪该级别及以上的事件

基础配置

logging.basicConfig是日志的基础配置，参数包括

参数名 描述
filename 日志文件名字
filemode 文件打开模式，默认为’a’，即追加模式，若想直接覆盖原文件，可改为’w’
format 消息输出的指定格式，详见LogRecord
datefmt 日期/时间格式，详见time.strftime()
style 不常用
level 指定记录级别，默认为logging.WARNING
stream 不常用
handlers 不常用

同时输入到文件和屏幕

默认的Hander是输出到文件的，因此需要添加一个Hander

多个文件中使用同一个logger

只要维护logger名字统一即可，然后每个模块中使用logging.getLogger(<name>) 获取对应的logger。
